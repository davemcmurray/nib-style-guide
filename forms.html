    • Forms

    We use forms to give customers the ability provide us with information.

    Forms matter because the failure of form results in failure demand or a loss sale. We don't like this stuff.

    So let's get forms right.

    The 4 c's of form design

    • Clear questions which have a clear and unambiguous answer.
    • Concise wording without fluff or jargon
    • Clever so that we offer help if a customer makes a mistake
    • Co-operative inputs so that even if the user strays from the "normal use case" their data can still be used.

    Section heading

    Usage

    Do

    • Use 'Roboto'
    • Use the field set to represent a set of form controls e.g ADDRESS DETAILS

    Example

    <h2 class="Title-2">ADDRESS DETAILS</h2>

    ADDRESS DETAILS


    CSS

    .title-2 {
    color: #444444;
    font-family: roboto;
    font-size: 36px;
    font-weight: 300;
    letter-spacing -1px;
    line-height: 48px;
    };

    Sub section heading

    Usage

    Do

    • Use 'Roboto'
    • Use a subheading when there is multiple groups of form controls within a section e.g ADDRESS section includes sub-section of Home address and Postal address

    Example

    <h3 class="Title-3">Home address</h3>

    Home address


    CSS

    .title-3 {
    color: #444444;
    font-family: roboto;
    font-size: 24px;
    font-weight: 300;
    letter-spacing -1px;
    line-height: 32px;
    }

    Labelling

    Usage

    Do

    • Use 'Open sans'
    • Prefer top left-aligned labels
    • Use uppercase
    • 14px
    • Use as few words as possible

    Do not

    • Use system speak - customers don't want to think like our database

    Example

    <label class="control-label">PHONE NUMBER</label>

    CSS

    .control-label {
    display: block;
    margin-bottom: 10px;
    color: #fff;
    font-size: 14px;
    font-weight: 700;
    text-transform: uppercase;
    };

    Text input

    Usage

    Do

    • Field size should match intended input length e.g phone number 10 characters
    • Preferably have validation on all fields however there are some exceptions e.g customer comments

    Example

    <input id="firstName" name="firstName" type="text" maxlength="24" class="control control-input control-input--text"></input>

    Radio buttons

    Usage

    Do

    • Used when there is a list of two or more options
    • Only one option can be selected
    • User must select an answer

    Example

    <input id="male">Male></input>

    CSS

    .control-label {
    display: block;
    margin-bottom: 10px;
    color: #444;
    font-size: 14px;
    font-weight: 700;
    text-transform: uppercase;
    }

    .control-label--radio {
    background-color: #00781c;
    background-color: rgba(0, 0, 0, 0.2);
    border-radius: 2px;
    -moz-box-sizing: border-box;
    box-sizing: border-box;
    color: #fff;
    color: rgba(255, 255, 255, 0.8);

    font-size: 16px;

    -webkit-transition: background 0.1s ease-in;
    -moz-transition: background 0.1s ease-in;

    }

    Select box

    Usage

    Do

    • Used when there are lists of options
    • User may select any number of choices including zero, one, or several

    Example

    <select class="control control-input control-input--select"></select>

    CSS

    .control-input--select {
    max-width: 100%;
    padding-right: 10px;
    color: #545454;
    cursor: pointer;
    }

    Checkbox

    Usage

    Do

    • Used when there are one or more options
    • Choosing to select is optional
    • Multiple options can be selected
    • Often used to acknowledge acceptence e.g Terms and Conditions

    Example:

    <p class="copy">Copy</p>
    We'll recognise any waiting periods already served with your current fund, so you can claim straight away. See our change your health insurance page. Any existing limits used with your current fund will carry across to nib.

    CSS

    .copy {
    color: #444444; font-family: open-sans;
    font-size: 14px;
    font-weight: 400;
    line-height: 20px;
    margin-bottom:
    };

    Alerts & Validation

    Usage

    Do

    General

    • Used to assist user to enter valid input
    • Help user as soon as possible e.g if wrong info entered let user know

    Valid input

    • Tick to be shown next to field on valid input

    Invalid input

    • Cross to be shown when input invalid
    • Alert/tool tip to be used to tell the user what they did wrong, and how to fix it

    Example:

    <p class="copy">Copy</p>
    We'll recognise any waiting periods already served with your current fund, so you can claim straight away. See our change your health insurance page. Any existing limits used with your current fund will carry across to nib.

    CSS

    .copy a {
    color: #009623;
    text-decoration: underline;
    };

    .copy a:hover {
    color:#26b11b;
    };

    Supporting information

    Usage

    Do

    • Used to inform user of required input e.g Password must be at least 8 characters and contain one uppercase letter and one icon

    Example:

    <p class="text-small">Disclaimer</p>
    text-small

    CSS

    .text-small {
    font-size: 12px; line-height: 18px;
    };
