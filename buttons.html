Buttons

Buttons, across the nib digital channels, help our users see and perform an action. These actions include:

  • Moving between pages
  • Submitting a form
  • Expanding or minimising an accordion
  • Opening a dialog

General button usage

Labelling

  • Open sans
  • Single line
  • 12px - 18px
  • Tell the user what the button will do
  • DO NOT use contextless request as labels e.g. "Click here"

Do

  • Button dos

Do not

  • Do not use multi-line line labels
  • Do not use modifiers that have not already been defined

Primary

Key attributes

  • Always either orange or green
  • Infrequency of use

Usage

Labelling

  • Uppercase
  • Bold

Location

  • Above the fold, where possible
  • Where the preceding content ends i.e. the conclusive action

Do

  • Use as main call to action
  • Use sparingly
  • Can be used with an icon e.g. New join indicating a step in the process
  • Can be used multiple times on a page when it performs the same action e.g. linking to quote on landing pages or selecting a product in Quote test

Do not

  • Use on the same page for different purposes e.g. nib Home page

Example

<button class="v2-button v2-button--primary">Click me!</button>

Modifiers

  • background-color:#ffb400;
  • background-color:#009623;
  • background-color:#888;
  • .full-width{width:100%}

Secondary

Key attributes

  • Has a background but is never orange
  • Accompanies primary

Usage

Labelling

  • Uppercase
  • Bold

Do

  • Use sparingly
  • Can be used with an icon e.g. No hospital button on Quote test

Do not

  • Use orange
  • Use more than 4 times on a single screen (to designer's discretion). If button is being used too frequently, consider using tertiary.

Example

<button class="v2-button v2-button--secondary">Click me!</button>

Modifiers

  • background-color:#009623;
  • background-color:#888;
  • .full-width{width:100%}

Tertiary

Key attributes

  • Visually lightweight
  • Used frequently
  • No background

Usage

Do

  • Use when actions or links are numerous on a screen
  • Can be used with an icon
  • Ensure it is obviously clickable. e.g. colour change, underline or icon

Do not

  • Use as a primary
  • Use a background

Example

<button class="v2-button v2-button--tertiary">Click me!</button>

Modifiers

  • color:#444;
  • color:#009623;
  • .full-width{width:100%}

Supplementary

Key attributes

  • Accompanies a button
  • Always a lesser call to action

Usage

Do

  • Use for an action that is an option but not preferred
  • Use in conjunction with higher buttons
  • Can be used with an icon e.g. New join indicating a step in the process
  • Can be used multiple times on a page when it performs the same action e.g. linking to quote on landing pages or selecting a product in Quote test

Do not

  • Use in conjunction with tertiary buttons

Example

<button class="v2-button v2-button--supplementary">Click me!</button>
